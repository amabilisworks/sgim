1. radio, checkbox id 값 수정시 반드시 label의 for 속성도 수정해주셔야 합니다.

예시

원본 코드

<input type="radio" class="ipt" name="rdoGroup_1" id="rdo0"><label for="rdo0">전체</label>

여기에 rdo_0을 의미있는 ID로 부여하기 위해 allSelect로 변경 한다면...

수정된 코드

<input type="radio" class="ipt" name="rdoGroup_1" id="allSelect"> <label for="allSelect">전체</label>

input의 ID 와 label의 for는 무조건 값이 같아야 합니다.

5. uiAct 클래스가 자꾸 생기는데...?

ui.init();가 실행되면 ui component에 맞게 스크립트가 실행되고
uiAct 클래스가 해당 element에 박히면 이것은 스크립트가 처리되어 정상 작동한다는 뜻이기도 합니다.
그래서 jsp파일에는 절대 uiAct라는 class로 하드코딩해서 사용해선 안됩니다.

7. a 태그의 href속성은 남겨주셔야 합니다.

눌렀을때 화면 튐현상 때문에 속성값 자체를 날리시는 분이 간혹 계신데 이는 접근성에 어긋나며 키보드 탭키로 접근이 되지 않습니다.

<a href="#">버튼</a> <script type="text/javascript"> $('button').on('click',function(e){ e.preventDefault(); 실행할 명령어 }); </script>

preventDefault(); 로 처리부탁드립니다.

8. radio, checkbox 에 click, onclick 이벤트는 사용하시면 안됩니다.

위 속성들은 키보드로만 값을 바꿀수 있기 때문에 click 이벤트를 사용하시면 키보드 동작시 이벤트가 발생하지 않습니다.

따라서 반드시 change 이벤트로 처리해주시고 checked 된 값을 체크해서 작업해주시길 바랍니다.
