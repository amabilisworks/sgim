원칙 지침 검사항목(지표) 오류유형 적용여부 비고
1. 인식의 용이성 1.1 대체 텍스트 1. 적절한 대체 텍스트 제공
(텍스트 아닌 콘텐츠는 그 의미나 용도를 이해할 수 있도록 대체 텍스트를 제공해야 한다.)

평가기준 :
의미 있는 텍스트 아닌 콘텐츠에 그 의미나 용도를 동등하게 인식할 수 있는 적절한 대체 텍스트를 제공했는지, 의미 있는 배경 이미지의 경우 그 의미가 사용자에게 전달 될 수 있도록 대체 텍스트를 제공했는지 평가한다.

 1-1 텍스트 아닌 콘텐츠에 대한 대체텍스트를 제공하지 않은 경우
  • 시각적으로 인지가 가능한 이미지 텍스트는 이미지와 동등한 정보를 제공하여야 함
  • alt 미정의 오류인 경우
  • 이미지맵의 img, area 요소에 alt 미정의 오류인 경우
  • longdesc 파일이 없는 경우
  • 의미 있는 이미지가 배경이미지로 제공되었으나 대체 콘텐츠가 제공되지 않은 경우
  • 특수문자 또는 기호 등으로 상승/하락, 상한/하한, 더보기 등의 정보를 제공한 경우 오류
  • 이미지 요소에 IR 기법 제공시 display:none; 또는 visibility:hidden; 속성을 사용한 경우
  • ASCII 심벌 기호를 이용하여 얼굴 형상을 표현하거나, 한글 자음 또는 모음을 나열하여 감정을 표현하는 문자 이모티콘을 심벌 또는 문자로만 나열하여 제공한 경우
 준수 고려사항 퍼블리싱 개발
  • 의미가 있는 이미지의 대체 텍스트가 빈값으로 제공되는 경우 오류
  • input type="image" 일 경우 alt="**" 대체 텍스트 필수
  • 열림/닫힘, 접기/펼치기 등에 대한 상태정보 제공 필수
  • QR 코드는 대체텍스트에 URL 제공 권장
1-2 텍스트 아닌 콘텐츠에 대한 대체 텍스트를 불충분 또는 적절하지 않게 제공한 경우
  • 대체텍스트의 정보가 불충분하게 제공된 경우(이미지 정보를 요약해서 의미만 전달하는 경우도 오류)
  • 대체텍스트가 오타 또는 업데이트가 되지 않은 경우
  • 장식 또는 꾸밈의 목적으로 제공된 의미 없는 이미지(또는 배경이미지)에 대체텍스트를 제공한 경우
  • 조직도, 도식 이미지, 차트 등 복잡한 이미지의 대체 텍스트는 단순 텍스트 제공 이상의 상호 관계, 수식, 흐름 등 충분한 설명을 제공해야 함.(HTML로 구조를 설명하고 있으나 잘못된 마크업으로 인해 구조를 이해할 수 없게 제공한 경우는 검사항목 1, 19 동시감점)
  • 이미지맵의 img 요소에 대체텍스트가 빈 값(alt="") 또는 잘못 제공된 경우
  • longdesc 연결되지 않은 경우
  • QR 코드, CCTV 등 실시간 영상 등에 대한 대체 텍스트는 콘텐츠의 용도나 목적을 알 수 있도록 제공.
  • 자동 가입 방지 이미지(CAPTCHA)에 대하여 대체 텍스트를 빈 값(Empty String)이나 동등한 텍스트 정보로 제공하지 않은 경우 음성 등의 대체 수단을 제공해야 함.(용도나 목적만 대체 텍스트를 제공하는 자동 가입 방지 이미지(CAPTCHA)에 대해서는 음성, 문자 안내 등의 대체 수단을 제공해야 함)
 준수
1-3 사용자가 등록한 텍스트 아닌 콘텐츠에 대해 관리자가 대체 텍스트 등록을 유도하는 안내 또는 대체 수단 입력 기능을 제공하지 않은 경우
  • 사용자가 등록한 텍스트 아닌 콘텐츠에 대체 텍스트 등록을 유도하는 안내 또는 대체 수단 입력 기능을 제공하지 않은 경우
 준수 고려사항 공통
  • 사용자가 등록한 텍스트 아닌 콘텐츠에 대해 대체 텍스트 입력 필드 제공
1-4 기타
  • alt와 title값을 동일하게 제공한 경우(수정권장)
  • 썸네일로 사용되는 단순 이미지가 해당 텍스트와 함께 링크에 포함될 경우 alt 중복설명 제공된 경우(수정권장)
  • 링크를 포함하며 텍스트 없이 단독으로 제공되는 이미지에 <a title="설명">은 제공되고 있으나 이미지의 alt를 빈 값으로 제공한 경우.
  • 플래쉬로 제작된 콘텐츠의 대체텍스트
 준수
1.2 멀티미디어 대체 수단 2. 자막 제공
(멀티미디어 콘텐츠에는 자막, 원고 또는 수화를 제공해야 한다.)

평가기준 :
멀티미디어 콘텐츠를 동등하게 인식할 수 있도록 자막, 원고 또는 수화등 대체 수단을 제공했는지 평가한다.

 2-1 멀티미디어 콘텐츠에 자막, 원고, 수화 중 하나 이상의 대체 수단을 제공하지 않은 경우
  • 멀티미디어 콘텐츠에 대한 대체수단을 제공하지 않은 경우
  • 음성이 없는 동영상(시각적 정보만 제공)에 대체수단이 없는 경우(ppt 슬라이드쇼 동영상 등)
  • 배경음만 재생되는 단순 동영상에 대체수단이 없는 경우.
  • 파일로 제공되는 영상, 음성 콘텐츠에 대체수단이 제공되지 않은 경우.
 준수 고려사항 공통
  • 멀티미디어 콘텐츠에 자막, 원고 또는 수화 제공 필수
2-2 멀티미디어 콘텐츠에 대한 대체 수단의 내용을 불충분 또는 적절하지 않게 제공한 경우
  • 멀티미디어 콘텐츠의 내용을 일부 누락하거나 요약하여 제공하여서는 안 됨
  • 멀티미디어 콘텐츠에 대한 대체 수단 제공 방식이 적절하지 않은 경우 오류
 준수 고려사항 공통
  • 멀티미디어 콘텐츠의 내용을 요약해서 제공하거나 자막 대신 콘텐츠에 대한 소개 내용만 제공한 경우 오류
2-3 사용자가 등록한 멀티미디어 콘텐츠에 대해 관리자가 대체 수단 등록을 유도하는 안내 또는 대체 수단 입력 기능을 제공하지 않은 경우
  • 멀티미디어 콘텐츠에 대한 대체 수단 제공 방식이 적절하지 않은 경우 오류
 준수 고려사항 공통
  • 멀티미디어 파일 첨부 시 자막을 업로드 하거나 생성할 수 있는 수단을 제공
1.3 명료성 3. 색에 무관한 콘텐츠 인식
(콘텐츠는 색에 관계없이 인식될 수 있어야 한다.)

평가기준 :
색을 배제하여도 인식할 수 있는 콘텐츠를 제공하고 있는지 평가한다.

 3-1 색상 만으로 내용을 구분하거나 인식하도록 제공된 콘텐츠(그래프, 차트, 지도, 필수 입력 항목 등)를 제공한 경우
  • 차트에 대한 데이터 테이블과 같이 대체 콘텐츠를 제공하지 않은 경우 오류
  • 사용자가 콘텐츠 선택 시 현재 선택된 정보를 색상만으로 구분한 경우 오류. (현재 선택된 정보를 색상 이외에 테두리, 배경, 패턴 등으로 구분해주어야 함.)
 준수 고려사항 공통
  • 그래프 및 차트 오픈 api 사용시 적용
  • 그래프의 경우 명도 대비로 구분이 가능하더라도 색상만으로 내용을 구분하도록 제공한 경우 오류
  • 현재 단계를 색상만으로 제공한 경우 오류. 다양한 방법으로 현재 단계 임을 알 수 있도록 제공(대체텍스트, 패턴 등)
  4. 명확한 지시 사항 제공
(지시 사항은 모양, 크기, 위치, 방향, 색, 소리 등에 관계없이 인식될 수 있어야 한다.)

평가기준 :
지시 사항 정보를 특정 감각에 의존하지 않고 다양한 감각을 통해 용도나 목적을 이해할 수 있도록 제공하는지 평가한다.

 4-1 색, 크기, 모양, 방향 등 지시사항을 시각정보로만 제공한 경우
  • 별도의 안내 내용 없이 시각정보로만 정보를 제공한 경우 오류
  • 별도의 안내 내용 없이 특수문자 등으로 필수입력 정보를 제공한 경우 오류
 준수 고려사항 디자인 기획
  • '웹 콘텐츠는 콘텐츠에 접근하는 사용자들이 색, 크기, 모양 또는 위치에 관한 정보를 인식하지 못하더라도 원하는 콘텐츠에 접근할 수 있도록 제작되어야 한다'
  • ex) 하단 원형 버튼을 누르세요.
  • ex) 위 박스 안의 붉은 글씨 내용을 참고하세요
  4-2 소리 등 지시사항을 청각정보로만 제공한 경우
  • 별도의 안내 내용 없이 청각정보로만 정보를 제공한 경우 오류
  5. 텍스트 콘텐츠의 명도 대비
(텍스트 콘텐츠와 배경 간의 명도 대비는 3대 1이상이어야 한다.)

평가기준 :
화면 확대가 가능하도록 구현한 콘텐츠의 모든 텍스트 콘텐츠(텍스트 및 텍스트 이미지 포함)는 명도 대비 3:1 이상 준수

 5-1 일반 텍스트 또는 이미지 텍스트의 명도 대비가 3:1을 만족하지 않은 경우
  • 화면 확대가 가능하도록 구현한 콘텐츠의 모든 텍스트 콘텐츠(텍스트 및 텍스트 이미지 포함)는 명도 대비 3:1 이상 준수
  • 텍스트의 역할을 하는 이미지나 특수 문자도 최대 3:1의 명도 대비를 만족해야 함
 준수 고려사항 디자인
  • 확대 축소가 가능한 경우(PC 브라우저) : 전경색과 배경색의 명도 대비가 3:1 이상을 만족하지 않으면 오류
    (최소 기준 - 배경색 #ffffff / 전경색 #959595)
  • 확대 축소가 불가능한 경우(모바일) : 명도 대비가 4.5:1 이상을 만족하지 않으면 오류
    (최소 기준 - 배경색 #ffffff / 전경색 #767676)
  6. 자동 재생 금지
(자동으로 소리가 재생되지 않아야 한다.)

평가기준 :
3초이상 자동으로 재생되는 배경음(동영상, 음성, 음악 등)을 제공하는지 평가한다.

단, 아래의 경우는 준수한 것으로 인정한다.
  • 음성 정보 없이 동영상의 자동 재생은 준수한 것으로 인정한다.
  • 웹 페이지 첫 부분에 배경음을 제어할 수 있는 수단을 제공할 경우 준수한 것으로 인정한다.
  • ESC 키를 눌러 멈춤 또는 음소거가 제공될 경우 준수한 것으로 인정한다.
 6-1 웹 페이지에서 자동적으로 재생되는 3초 이상의 배경음 콘텐츠를 제공하는 경우
  • 웹 페이지에서 자동적으로 재생되는 3초 이상의 배경음 콘텐츠를 제공하는 경우
  • 자동 배경음 외에도 마우스 오버 또는 키보드 초점만으로 3초 이상 배경음 콘텐츠가 재생되는 경우
 준수
  7. 콘텐츠 간의 구분
(이웃한 콘텐츠는 구분될 수 있어야 한다.)

평가기준 :
다음과 같이 이웃한 콘텐츠를 이용할때 구분되도록 제공하는지 평가한다.

  • 테두리를 이용하여 구분함
  • 콘텐츠 사이에 시각적인 구분선을 삽입하여 구분함
  • 서로 다른 무늬를 이용하여 구분함
  • 콘텐츠 배경색 간의 명도대비(채도)를 달리하여 구분함
  • 줄 간격 및 글자 간격을 조절하여 구분함
  • 기타 콘텐츠를 시각적으로 구분할 수 있는 방법 등을 이용하여 구분함
 7-1 이웃한 콘텐츠를 1가지 방법 이상으로 구분하지 않은 경우
  • 이웃한 콘텐츠는 서로 같은 수준의 콘텐츠로 저 시력 사용자도 구분할 수 있을 정도로 구분해주는 것을 권장
  • 테두리, 구분 선, 콘텐츠 배경 색 간의 명도 대비, 기타 시각적으로 구분할 때 구분을 위한 색과 배경색의 명도 대비가 4.5:1 이상이 되도록 하는 것을 권장
 권장 고려사항 디자인
  • 시각적으로 콘텐츠 구분이 불가능 한 경우에 한해서 제한적으로 오류
2.운용의 용이성 2.1 입력장치 접근성 8. 키보드 사용 보장
(모든 기능은 키보드만으로도 사용 할 수 있어야 한다.)

평가기준 :
모든 기능을 키보드로 접근 및 운용 가능한지 평가한다.

 8-1 마우스로 제어할 수 있는 요소를 키보드로 제어할 수 없는 경우
  • 마우스 이벤트만 적용된 경우
  • 이미지 또는 초점을 받을 수 없는 요소에 onclick 이벤트가 적용된 경우(검사항목 8, 9 동시감점)
  • 이미지 또는 초점을 받을 수 없는 요소에 마우스 이벤트만 적용된 경우(검사항목 8, 9 동시감점)
  • a 요소에 href 속성이 제공되지 않아 키보드 접근 및 초점이 표시되지 않는 경우(검사항목 8, 9 동시감점)

  • 스크립트 사용으로 인한 비 논리적 이동 및 함정은 오류

 17-1 문서 타입에 맞는 기본 언어를 명시하지 않은 경우
  • Doctype에 기본 언어 표시 속성이 지정되지 않은 경우
 준수 고려사항 퍼블리싱 개발
  • 기본 언어 표시는 Doctype에 맞는 속성을 사용하여 ISO 639-1에서 지정한 두 글자로 된 언어 코드를 사용해야 함, 기본 언어를 제공하지 않은 경우 오류.
    • HTML 4.01 / HTML 5 : <html lang="ko" ... >
    • XHTML 1.0 : <html xml:lang="ko" lang="ko" ... >
    • XHTML 1.1 : <html xml:lang="ko" ... >
  • 페이지 내 언어가 바뀌는 경우 시작 요소에서 언어를 변경하여 명시해 줄 것을 권장.
17-2 문서 타입에 맞는 기본 언어를 적절하지 않게 명시한 경우
  • 페이지 내 언어가 바뀌는 경우
 권장
  3.2 예측 가능성 18. 사용자 요구에 따른 실행
(사용자가 의도하지 않은 기능(새 창, 초점 변화 등)은 실행되지 않아야 한다.)

평가기준 :
사용자가 의도하지 않은 기능(새창, 초점 변화)을 제공하는 지 평가한다.

 18-1 사용자가 실행하지 않은 상황에서 예측하지 않은 새 창이 열리는 경우
  • 사용자가 예측하지 못한 새 창이 열리는 경우
 준수 고려사항 퍼블리싱 개발
  • 사용자가 예측하지 못한 상황에서 새창이 제공되는 경우 오류.
  • <a target="_blank">로만 새 창을 알린 경우는 준수(title="새창열림" 없어도 됨)
  18-2 웹 페이지 시작 시, 새 창 또는 화면을 가리는 레이어 팝업을 제공하는 경우
  • 페이지 로딩 시, 레이어 팝업 또는 모달창이 제공되는 화면을 가리는 경우
 준수 고려사항 공통
  • 웹 페이지 시작 시, 새 창 또는 화면을 가리는 레이어 팝업을 제공하는 경우 오류.
  • 단, 은행·증권사등 고객의 돈(이익)과 관련된 긴급 공지는 허용.
  18-3 사용자가 의도하지 않은 초점 변화나 기능이 발생되어 맥락상 불편을 주는 경우
  • 초점이 갑자기 사라지게 되는 경우(검사항목 8, 9, 18 동시감점)
  • 팝업창에서 ‘다시 보지 않기’ 등의 확인란 선택으로 자동 서식이 제출되어, 팝업이 사라지는 경우
  • 요소박스에서 데이터 입력 시, 다음 영역으로 자동 이동 하는 경우
  • select에 onchange를 사용하여 select의 option 값을 선택하는 것만으로 페이지가 새롭게 변경되는 경우(검사항목 8, 18 동시감점)
 준수 고려사항 개발
  • 텍스트 입력 서식 간 자동 초점 이동 기능(autofocus)은 역방향(Shift+Tab) 미적용시 오류.
  • 오류정정을 위한 자동 초점 변경은 필수
  3.3 콘텐츠의 논리성 19. 콘텐츠의 선형 구조
(콘텐츠는 논리적인 순서로 제공해야 한다.)

평가기준 :
CSS 제거후 콘텐츠의 순서가 논리적으로 선형화 되었는지 평가한다.

 19-1 콘텐츠의 순서가 논리적으로 제공되지 않아 계층 구조를 이해할 수 없는 경우
  • [제목-내용] 콘텐츠 목록의 배치가 분리되어 이해가 불가능한 경우
  • 기능적인(로그인, 검색 등) 항목의 순서가 논리적이지 않게 제공한 경우
  • 사용방법 안내 정보를 콘텐츠 이전에 제공하지 않은 경우 (수정 권장)
 준수 고려사항 퍼블리싱 개발
  • 텍스트 입력 서식의 부연 설명은 시각적으로는 입력 서식 옆이나 뒤지만 논리적 구조 상에서는 부연 설명이 텍스트 입력 서식 앞에 와야 한다.
  20. 표의 구성
(표는 이해하기 쉽게 구성해야 한다.)

평가기준 :
표를 이해할 수 있도록 정보를 제공하고 있는지 평가한다.

  • <caption> 표의 제목
  • <th>와 <td> 제목 셀 구분
 20-1 표의 제목과 요약 정보를 제공하지 않은 경우 준수 고려사항 퍼블리싱 개발
  • <caption> 제목과 summary 속성을 제공하지 않았을 경우 오류.
  20-2 표의 제목과 요약 정보가 불충분하거나 적절하게 제공되지 않은 경우 준수 고려사항 퍼블리싱 개발
  • <caption> 제목과 summary 속성을 동일하게 제공한 경우 오류.
  • html5는 <caption>에 summary 내용 추가 권장.
  20-3 제목 셀과 내용 셀을 구분하지 않거나 잘못 구분한 경우 준수 고려사항 퍼블리싱 개발
  • 데이터 테이블에 <th> 요소를 사용하여 제목 셀을 제공하지 않은 경우 오류.
  • 첫 행과 첫 열이 제목 셀인 경우 scope 속성을 사용하여 행 제목인지 열 제목인지 구분.
  • 제목 셀이 첫 행 또는 첫 열만 있을 경우 scope 속성 생략 가능.
  20-4 다단, 병합 등 복잡한 표를 제공 시 headers, id 속성을 통해 제목 셀과 내용 셀을 연결하지 않은 경우 준수 고려사항 퍼블리싱 개발
  • 가로 퐁당퐁당 데이터 테이블 올바른 예
    <tr>
      <th id="test1">제목 1</th>
      <td headers="test1">내용 1</td>
      <th id="test2">제목 2</th>
      <td headers="test2">내용 2</td>
    </tr>
  20-5 표의 정보를 이해하기 어려운 중첩 표를 넣어 정보를 제공한 경우 준수 고려사항 퍼블리싱 개발
  • 데이터 테이블에 중첩 표를 사용한 경우 오류.
  • 읽기 게시판의 콘텐츠에 데이터 테이블이 올 가능성이 있는 경우 구조를 레이아웃용 테이블 또는 다른 요소로 마크업 할것.
  • 레이아웃용 표테이블의 경우 caption, thead, tfoot, th 등의 데이터테이블 요소로 구성되면 안됨
  3.4 입력 도움 21. 레이블 제공
(입력 서식에는 대응하는 레이블을 제공해야 한다.)

평가기준 :
입력 서식에 대응하는 레이블을 제공하는지 평가한다.

 21-1 각 입력 서식에 1:1로 대응하는 레이블을 제공하지 않은 경우 준수 고려사항 퍼블리싱 개발
  • <input type="image | hidden | submit | button | reset">을 제외한 모든 <input>, <textarea>, <select> 요소에 1:1 대응하는 <label>요소 또는 title 속성을 통해 레이블 제공
  • <select> 요소의 첫 번째 <option>이 레이블 역할을 대신하는 경우라도 <label> 또는 title 속성을 통해 레이블 제공
  • 약관/로그인 화면/검색 구성시 fieldset 사용
  21-2 각 입력 서식에 1:1로 대응하는 레이블을 적절하게 제공하지 않은 경우 준수 고려사항 퍼블리싱 개발
  • 1:1로 대응 가능한 레이블이 있음에도 title 속성으로 이를 제공한 경우 오류.
  22. 오류 정정
(입력 오류를 정정할 수 있는 방법을 제공해야 한다.)

평가기준 :
입력 오류를 정정할 수 있는 방법을 제공하는지 평가한다.

 22-1 입력 오류 발생 시, 오류 원인 및 내용을 알려주지 않거나 적절하지 않은 경우 준수
  22-2 입력 오류 발생 시, 정정할 수 있는 수단을 제공하지 않거나 적절하지 않은 경우 준수
  22-3 오류 정보 확인 시, 해당 입력 서식으로 초점이 이동하지 않는 경우 준수 고려사항 공통
  • 오류 정정에 대한 정보 제공 후 오류가 난 입력 서식으로 초점이 이동할 수 있도록 링크나 기능 제공 필수
  22-4 입력 오류 발생 시 입력 내용이 모두 사라지는 경우 준수 고려사항 개발
  • 오류 메시지 창에서 확인을 누른 후 입력한 글 내용이 모두 사라는 경우 오류
4.견고성 4.1 문법 준수 23. 마크업 오류 방지
(마크업 언어의 요소는 열고 닫음, 중첩 관계 및 속성 선언에 오류가 없어야 한다.)

평가기준 :
마크업 언어 요소의 열고 닫음, 중첩 관계 및 속성 선언에 오류 없이 제공하는지 평가한다.

 23-1 태그의 열고 닫음 오류 준수 고려사항 퍼블리싱 개발
  • 시작 태그 또는 종료 태그를 누락한 경우 오류.
  • http://validator.w3.org 검수
  • 파이어폭스 또는 크롬 부가기능 활용.
    (Firebug, Html Validator 사용)
  23-2 태그의 중첩 오류 준수 고려사항 퍼블리싱 개발
  • 태그 중첩를 잘못한 경우 오류.
  23-3 중복 선언된 속성 오류, id 값 중복 사용 오류 준수 고려사항 퍼블리싱 개발
  • id 값이 중복으로 제공된 경우 오류.
  • 속성이 중복으로 제공된 경우 오류.
  23-4 웹 표준 문법에 맞지 않는 경우 준수
  4.2 웹 애플리케이션 접근성 24. 웹 애플리케이션 접근성 준수
(콘텐츠에 포함된 웹 애플리케이션은 접근성이 있어야 한다.) 		24-1 웹 애플리케이션이 자체적인 접근성이 없으며 사용자가 선택할 수 있는 대체 콘텐츠가 존재하지 않거나 적절하지 않게 제공한 경우 준수
